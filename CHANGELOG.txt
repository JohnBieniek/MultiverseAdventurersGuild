Multiverse Adventurers Guild Site Changelog

This changelog summarizes the major content, rules, formatting, parser, and layout updates made while revising the site from the older public version at https://www.multiverseadventurersguild.com/ into the current project state.


Core Site Structure and Navigation

- Renamed the reusable guide renderer from GuidePage to Guide because it is a component, not a page.
- Renamed src/components/GuidePage.jsx to src/components/Guide.jsx.
- Updated Players, GM, and Rules pages to import and render the Guide component.
- Renamed the CSS wrapper class from guide-page to guide.
- Updated the related CSS comment from "Guide Pages" to "Guide component."
- Removed stale GuidePage references from the source.
- Changed the browser tab icon to use the G icon from public assets.
- Tightened guide header spacing so Players, Rules, and GM headers align more consistently with inner section content.
- Reduced page margins and sidebar whitespace at desktop sizes so reference pages have more usable reading space.
- Adjusted wide-screen content widths for Players and other guide pages so text and subheader buttons fit better.
- Tuned huge-resolution widths so pages do not become too wide on very large displays.


Guide Parser and Reference Linking

- Removed archetype and talent alias maps from the Guide component.
- References now link only when the displayed name matches the actual card name.
- Fixed content references instead of relying on aliases.
- Updated Halfling Strengths from Stealth to Sneak so it matches the actual skill name.
- Changed Screamer from Preferred Talent to Preferred Talents so it parses like the other archetypes.
- Fixed contact parsing so Category, Example Name, and Expertise lines do not become separate cards.
- Tightened contact card detection so a contact must use the pattern Name - description, followed by Category and Example Name.
- Fixed the contact parser so Mechanic can be a contact name even though Mechanic is also a field label in talent cards.
- Verified Locksmith and Mechanic render as separate contact cards.
- Added support for automatic Fighting Styles grouping in the Weapons section based on Category: Fighting Style.


Responsive Layout and Visual Design

- Reduced guide page margins at desktop sizes to provide more room for content.
- Reduced sidebar padding and gaps, especially around the Players page sidebar.
- Adjusted subnavigation button sizing for 1920px, below 1920px, above 1920px, and very large screens.
- Restored smaller subnavigation button sizing at 1920px and lower after widening page content.
- Increased subnavigation button sizing only above 1920px.
- Changed footer sizing at wide resolutions.
- Made weapon cards display multiple per row on desktop:
  - 1 column on mobile.
  - 2 columns on desktop.
  - 3 columns on wider desktop.
- Added a Fighting Styles header before Fighting Style weapon cards.
- Changed the Fighting Styles header color from red to black.
- Set Contact cards to 1 column at 1920px and below.
- Set Contact cards to 4 columns above 1920px.
- Set Reputation cards to 3 columns on desktop and 4 columns above 1920px.
- Made Archetype cards 2 columns on very wide monitors above 1920px.
- Tried and then reverted multi-column Talent card layout on ultra-wide monitors.
- Capped Buff Option cards to a maximum of 3 columns.
- Adjusted Buff Option columns so they use the full available width rather than becoming skinny columns.
- Changed card field labels such as Description, Mechanic, Minor Version, Major Version, Category, Example Name, and Expertise from red to belt gold, then changed them again to the title-bar green for better readability.
- Kept Buff Option names such as Genius, Pathfinder, and Silver Tongue in red.
